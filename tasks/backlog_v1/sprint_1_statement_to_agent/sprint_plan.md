# Sprint 1 交割单到 Agent 建模

## Sprint 目标
完成从交割单上传、状态流转、文件识别、解析标准化，到 Profile 提取与 Agent 创建的业务闭环。

## 做什么
- [ ] 打通真实上传入口、状态机和失败处理
- [ ] 将原始交割单解析为标准化 TradeRecord
- [ ] 基于标准化交易记录提取 Profile 并创建 Agent

## 不做什么
- [ ] 不实现市场世界、撮合、账本和日循环
- [ ] 不实现前端观察台和排行榜
- [ ] 不实现 OpenClaw 接入

## Epic 总览
| Epic | Story 数 | 核心职责 |
| :--- | :--- | :--- |
| 交割单摄入 | 4 | 完成上传 API、状态机、文件类型识别和失败回滚。 |
| 交割单解析与标准化 | 3 | 把原始交割单映射成标准化 TradeRecord，并生成解析报告。 |
| Agent DNA / Profile | 3 | 从标准化交易记录提取画像、生成风格标签和 Agent。 |
