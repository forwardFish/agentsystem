# Sprint 4 Agent 广场与群体运营

## Sprint目标
让平台从单 Agent 演示升级为多 Agent 持续运行与可比较的展示平台。

## 不做什么
- [ ] 不实现社交评论与关注系统
- [ ] 不实现复杂推荐算法
- [ ] 不实现多租户商业化治理

## Epic 总览
| Epic | Story数 | 核心职责 |
| :--- | :--- | :--- |
| 种子 Agent 与批量供给 | 2 | 解决平台冷启动，确保一进入平台就能看到一批可运行 Agent。 |
| 广场与发现 | 4 | 为大量 Agent 提供列表、筛选、浏览与只读公共展示能力。 |
| 比较与榜单 | 2 | 让围观者能在统一世界里横向比较多个 Agent。 |
| 运行调度与生命周期 | 2 | 让很多 Agent 能持续运行、可暂停、可恢复、可归档。 |
